**Project:** Sample Space Expansion Detector (SSED)
**Course:** MGMT 69000 — Mastering AI for Finance, Purdue University
**Framework:** DRIVER (Discover → Define → Represent → Implement → Validate → Evolve)
**Tool:** Claude Code (claude-opus-4-6 / claude-sonnet-4-6)

## Summary: Where Human Judgment Was Required

The AI generated structure, code, and drafts throughout. These were the decisions that required human judgment — the ones Claude couldn't make without direction:

1. **Thesis framing** — The P vs X distinction is original intellectual work. Claude confirmed it lacked precedent; I defined what it meant and why it mattered.
2. **What NOT to build** — Excluded Reddit/PRAW, FinBERT, LangChain, and Assistants API based on signal-to-complexity tradeoffs.
3. **Default demo configuration** — Set `NVDA/CHGG/SPY, 2022-11-30` as the zero-argument demo case. A tool that needs explanation before it runs is not a demo.
4. **Bug prioritization** — Identified the three Validate-phase bugs, specified the fix order, and reviewed each fix before approving the next.
5. **CI failure diagnosis** — Read 4 separate CI error logs, diagnosed root causes (unused imports, INI parsing, missing pytest install, degenerate HMM input), and directed specific fixes.
6. **API migration decision** — Diagnosed financialdatasets.ai as a free-tier limitation (not an integration error), confirmed yfinance as the correct fallback, and updated documentation to accurately reflect both.
7. **UI placement decisions** — Specified that AI interpretation leads the page, non-determinism labels appear above results, and context-aware chat replaces two redundant panels.
